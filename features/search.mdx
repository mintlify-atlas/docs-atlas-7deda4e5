The search functionality in Cinemusic allows you to find songs, filter by artists, and browse by music genres with real-time filtering.

## Search architecture

The search system uses a combination of search types and state management:

- **Search types**: Songs, artists, and categories (genres)
- **Real-time filtering**: Updates as you type
- **State management**: Angular signals for reactive updates

## Search types

The application supports three search types:

```typescript
export enum TypeSearchEnum {
  CATEGORY = "CATEGORY",
  SEARCH = "SEARCH",
  ARTIST = "ARTIST",
}
```

### Search data model

```typescript
export interface TypeSearch {
  type: String;
  value: String;
}
```

## Search state management

The `StateSectionSearchService` manages search state:

```typescript
export class StateSectionSearchService {
  public dataSearch: WritableSignal<TypeSearch> = signal<TypeSearch>({
    type: TypeSearchEnum.SEARCH,
    value: "ALL"
  });
  public activationSong: WritableSignal<boolean> = signal<boolean>(true)
  public activationGender: WritableSignal<boolean> = signal<boolean>(false);
  public activationSinger: WritableSignal<boolean> = signal<boolean>(false);

  public setStateElements(data: string): void {
    if (data === TypeSearchEnum.ARTIST) {
      this.activationSong.set(false);
      this.activationGender.set(false);
      this.dataSearch.set({type: TypeSearchEnum.ARTIST, value: "ALL"})
    } else if (data === TypeSearchEnum.CATEGORY) {
      this.activationSong.set(false);
      this.activationSinger.set(false);
      this.dataSearch.set({type: TypeSearchEnum.CATEGORY, value: "ALL"})
    } else {
      this.activationSinger.set(false);
      this.activationGender.set(false);
      this.dataSearch.set({type: TypeSearchEnum.SEARCH, value: "ALL"})
    }
  }
}
```

<Note>
The search state tracks which search type is active and updates the UI accordingly.
</Note>

## Implementing search

### Search component

The `SectionSearchComponent` handles search input and filtering:

```typescript
export class SectionSearchComponent {
  private stateMusicService: StateMusicService = inject(StateMusicService);
  protected dataSearch: WritableSignal<TypeSearch> = 
    this.stateMusicService.stateSectionSearchService.dataSearch;

  protected onChangeInput(value: string): void {
    this.dataSearch.set({
      type: this.dataSearch().type,
      value: value
    })

    let listDataFiltered: Song[] = 
      this.stateMusicService.stateSectionMainMusicService.changeFilteredSongs(this.dataSearch());

    if (listDataFiltered.length > 0) {
      this.stateMusicService.stateMusicPlayerService.listSongs.set(listDataFiltered);
    }
  }
}
```

## Search activation states

The search component maintains three activation states:

```typescript
public activationSong: WritableSignal<boolean> = signal<boolean>(true)
public activationGender: WritableSignal<boolean> = signal<boolean>(false);
public activationSinger: WritableSignal<boolean> = signal<boolean>(false);
```

Only one search type can be active at a time, ensuring a clear user experience.

## Integration with music player

When search results are available, they're automatically loaded into the music player:

```typescript
if (listDataFiltered.length > 0) {
  this.stateMusicService.stateMusicPlayerService.listSongs.set(listDataFiltered);
}
```

This allows users to:
- Play filtered results immediately
- Navigate through search results using player controls
- Maintain playback state while searching

## Best practices

<Info>
**Performance**: The search filtering happens in real-time but only updates the player when results are available, preventing empty playlist states.
</Info>

<Note>
**Type safety**: Using TypeScript enums for search types and categories ensures type safety and prevents invalid search states.
</Note>
