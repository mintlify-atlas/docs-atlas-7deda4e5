## Basic icon components

Most icon components share the same basic interface:

### Props

<ParamField path="width" type="string" default="varies">
  The width of the icon. Accepts any valid CSS width value.
</ParamField>

<ParamField path="height" type="string" default="varies">
  The height of the icon. Accepts any valid CSS height value.
</ParamField>

<ParamField path="color" type="string" default="varies">
  The fill color of the icon. Accepts color names, hex values, or CSS variables.
</ParamField>


### Props

<ParamField path="width" type="string" default="'100%'">
  The width of the heart icon.
</ParamField>

<ParamField path="height" type="string" default="'100%'">
  The height of the heart icon.
</ParamField>

<ParamField path="isFavorite" type="boolean" default="false">
  When `true`, displays the heart in the favorite state (primary color). When `false`, displays in the not-favorite state (white).
</ParamField>

### Usage examples

```html
<!-- Not favorited -->
<app-svg-heart 
  width="1rem" 
  height="1rem" 
  [isFavorite]="false" 
/>

<!-- Favorited -->
<app-svg-heart 
  width="1rem" 
  height="1rem" 
  [isFavorite]="true" 
/>

<!-- With click handler -->
<app-svg-heart 
  (click)="toggleFavorite(song)" 
  width="1rem" 
  height="1rem" 
  [isFavorite]="song.isFavorite" 
/>
```

## Common patterns

### Using icons with buttons

```html
<button (click)="play()">
  <app-svg-play width="1rem" height="1rem" color="white" />
  Play
</button>
```

### Responsive icon sizes

```html
<!-- Percentage-based sizing -->
<div class="icon-container">
  <app-svg-play width="20%" height="20%" color="black" />
</div>

<!-- Fixed sizing -->
<app-svg-play width="1.2rem" height="1.2rem" color="white" />
```

### Using CSS variables for colors

```html
<app-svg-play width="13%" height="13%" color="var(--primary-color)" />
<app-svg-heart width="100%" height="100%" [isFavorite]="isLiked" />
<app-svg-search color="var(--color-gray-white)" colorHover="var(--primary-color)" />
```

## TypeScript integration

```typescript
import { Component, signal } from '@angular/core';
import { SvgPlayComponent } from './ui/components/svg-play/svg-play.component';
import { SvgHeartComponent } from './ui/components/svg-heart/svg-heart.component';

@Component({
  selector: 'app-song-card',
  imports: [SvgPlayComponent, SvgHeartComponent],
  template: `
    <div class="song-card">
      <button (click)="play()">
        <app-svg-play width="2rem" height="2rem" color="white" />
      </button>
      <app-svg-heart 
        (click)="toggleFavorite()" 
        width="1.5rem" 
        height="1.5rem" 
        [isFavorite]="isFavorite()" 
      />
    </div>
  `
})
export class SongCardComponent {
  isFavorite = signal(false);

  play() {
    console.log('Playing song...');
  }

  toggleFavorite() {
    this.isFavorite.update(val => !val);
  }
}
```

## Best practices

- Use consistent sizing across similar UI elements (e.g., all toolbar icons at `1.5rem`)
- Leverage CSS variables for colors to maintain consistent theming
- Add click handlers directly on the icon components for interactive icons
- Use percentage-based sizing for responsive icons within containers
- Use rem-based sizing for consistent absolute sizing
- Apply the `isFavorite` prop to the heart icon to reflect state
