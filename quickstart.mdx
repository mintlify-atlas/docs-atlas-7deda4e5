## Prerequisites

Before starting, make sure you have completed the [installation](/installation) steps. You should have:

- Node.js and npm installed
- Angular CLI installed globally
- Cinemusic repository cloned
- All dependencies installed via `npm install`

## Start the development server

Let's get Cinemusic running on your local machine.

<Steps>
  <Step title="Navigate to the project directory">
    Open your terminal and navigate to the Cinemusic directory:
    
    ```bash
    cd cinemusic
    ```
  </Step>
  
  <Step title="Start the development server">
    Run the Angular development server:
    
    ```bash
    ng serve
    ```
    
    Or use the npm script:
    
    ```bash
    npm start
    ```
    
    <Info>
      Both commands do the same thing. The `npm start` command is a shortcut defined in `package.json` that runs `ng serve`.
    </Info>
    
    You should see output similar to:
    
    ```
    ** Angular Live Development Server is listening on localhost:4200 **
    ✔ Browser application bundle generation complete.
    ```
  </Step>
  
  <Step title="Open the application">
    Open your browser and navigate to:
    
    ```
    http://localhost:4200/
    ```
    
    The application will automatically reload if you make changes to the source files.
  </Step>
</Steps>

<Note>
  The first compilation may take a minute. Subsequent rebuilds will be much faster thanks to Angular's incremental compilation.
</Note>

## Explore the application

Once the server is running, you can explore Cinemusic's features:

### Music section

The default route redirects to the music section at `/music`. Here you can:

- Browse music organized in a bento grid layout
- View albums, playlists, and categories
- Create custom playlists
- Use the integrated music player to play songs

### Navigation

The application includes several key routes:

| Route | Description |
|-------|-------------|
| `/music` | Main music browsing interface |
| `/music/myMusic` | Your personal playlists and songs |
| `/series` | Series and content sections |
| `/series/gender/:gender` | Filter series by genre |
| `/series/serie/:id` | View specific series details |

### Multilingual support

Cinemusic supports multiple languages out of the box:

<Steps>
  <Step title="Access language settings">
    Look for the language selector in the application interface (typically in the header or settings area).
  </Step>
  
  <Step title="Switch languages">
    Toggle between available languages (Spanish is the default, with English support).
    
    The application uses **ngx-translate** to load translations from `/src/i18n/` JSON files.
  </Step>
</Steps>

## Development commands

Here are the essential commands you'll use during development:

<CodeGroup>

```bash Development server
# Start the dev server (default port 4200)
ng serve

# Start on a custom port
ng serve --port 4300

# Start with production configuration
ng serve --configuration production
```

```bash Building
# Build for production
ng build

# Build and watch for changes
ng build --watch --configuration development

# Build output is in dist/cinemusic/
```

```bash Testing
# Run unit tests with Karma
ng test

# Run tests in headless mode (for CI/CD)
ng test --browsers=ChromeHeadless --watch=false
```

```bash Deployment
# Deploy to GitHub Pages
ng deploy

# Or use the npm script
npm run deploy
```

</CodeGroup>

## Configuration files

Key configuration files you should know about:

### app.config.ts

The main application configuration at `src/app/app.config.ts` sets up:

```typescript
export const appConfig: ApplicationConfig = {
  providers: [
    provideZoneChangeDetection({ eventCoalescing: true }), 
    provideRouter(routes),
    provideHttpClient(withInterceptors([errorInterceptor, mapperInterceptor])),
    provideAnimationsAsync(),
    provideTranslateService({
      defaultLanguage: 'es',
      loader: {
        provide: TranslateLoader,
        useFactory: httpLoaderFactory,
        deps: [HttpClient]
      }
    })
  ]
};
```

This configuration includes:
- Router with lazy-loaded routes
- HTTP client with custom interceptors for error handling and data mapping
- Browser animations
- Translation service with Spanish as the default language

### angular.json

The `angular.json` file configures:
- Build output path: `dist/cinemusic`
- Base href: `/cinemusic/` (for GitHub Pages deployment)
- Asset management and styles
- Build budgets and optimizations

## Troubleshooting

### Server won't start

If `ng serve` fails:

1. Ensure all dependencies are installed: `npm install`
2. Clear the Angular cache: `rm -rf .angular/cache`
3. Check that port 4200 is not in use by another process

### Blank page in browser

If you see a blank page:

1. Open browser developer tools (F12) and check the Console for errors
2. Verify the server is running on port 4200
3. Try accessing `http://localhost:4200/music` directly

### Translation files not loading

If translations don't appear:

1. Check that `src/i18n/` contains `es.json` and `en.json` files
2. Verify the translation files are valid JSON
3. Ensure the HTTP loader is configured correctly in `app.config.ts`
