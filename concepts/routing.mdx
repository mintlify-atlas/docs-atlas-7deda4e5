Cinemusic uses Angular's built-in router to handle navigation between different sections of the application. All routes are configured with lazy loading to optimize bundle size and initial load time.

## Route structure

The application has two main feature routes: `music` and `series`, with a default redirect to music.

```typescript app.routes.ts
import { Routes } from '@angular/router';

export const routes: Routes = [
    {
        path: 'music',
        loadComponent: () => import('./ui/pages/music/music.component').then((c) => c.MusicComponent),
        children: [
            {
                path: 'myMusic',
                loadComponent: () => import('./ui/pages/music/section-play-list-songs/section-play-list-songs.component').then((c) => c.SectionPLayListSongsComponent)
            }
        ]
    },
    {
        path: 'series',
        loadComponent: () => import('./ui/pages/series/series.component').then((c) => c.SeriesComponent),
        children: [
            {
                path: "gender/:gender",
                loadComponent: () => import('./ui/pages/series/section-gender-series/section-gender-series.component').then((c) => c.SectionGenderSeriesComponent)
            },
            {
                path: "serie/:id",
                loadComponent: () => import('./ui/pages/series/serie-selected-view/serie-selected-view.component').then((c) => c.SerieSelectedViewComponent)
            }
        ]
    },
    {
        path: "**",
        redirectTo: "music",
        pathMatch: "full"
    }
];
```

## Lazy loading

All routes use Angular's `loadComponent` function for lazy loading, which provides several benefits:

<AccordionGroup>
  <Accordion title="Reduced initial bundle size">
    Components are only loaded when the user navigates to their route, not at application startup.
    
    This significantly reduces the initial JavaScript bundle that needs to be downloaded and parsed.
  </Accordion>
  
  <Accordion title="Faster initial load time">
    By splitting the application into smaller chunks, the initial page load is much faster.
    
    Users can start interacting with the app while other features load in the background.
  </Accordion>
  
  <Accordion title="Improved performance">
    Each feature is loaded on-demand, reducing memory usage and improving overall application performance.
    
    Unused features never get loaded, saving bandwidth and resources.
  </Accordion>
</AccordionGroup>

### How lazy loading works

Instead of importing components directly:

```typescript
// ❌ Eager loading - component loaded at startup
import { MusicComponent } from './ui/pages/music/music.component';

export const routes: Routes = [
  { path: 'music', component: MusicComponent }
];
```

Cinemusic uses dynamic imports:

```typescript
// ✅ Lazy loading - component loaded on demand
export const routes: Routes = [
  {
    path: 'music',
    loadComponent: () => import('./ui/pages/music/music.component')
      .then((c) => c.MusicComponent)
  }
];
```

<Note>
  Angular automatically creates separate JavaScript bundles for each lazy-loaded route during the build process.
</Note>

## Default route

The application includes a wildcard route that redirects any unknown paths to the music section:

```typescript
{
  path: "**",
  redirectTo: "music",
  pathMatch: "full"
}
```

This ensures users always see valid content, even if they navigate to a non-existent URL.

## Navigation in components

To navigate between routes in your components, use Angular's Router service:

```typescript
import { Router } from '@angular/router';
import { inject } from '@angular/core';

export class MyComponent {
  private router = inject(Router);
  
  navigateToMusic() {
    this.router.navigate(['/music']);
  }
  
  navigateToSeries(genreId: string) {
    this.router.navigate(['/series/gender', genreId]);
  }
  
  navigateToSeriesDetail(seriesId: string) {
    this.router.navigate(['/series/serie', seriesId]);
  }
}
```
